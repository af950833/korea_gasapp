# Korea Gas App for Home Assistant

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-orange.svg)](https://github.com/hacs/integration)

Korea Gas App(가스앱)을 Home Assistant에서 사용할 수 있게 하는 커스텀 통합입니다.

SMS 본인인증으로 로그인하고, 가스 계약의 청구금액·검침 정보를 센서로 가져옵니다. 매일 오전 8시에 데이터를 갱신하며, 자가검침 기간이 되면 지정한 엔티티의 값을 읽어 자동으로 자가검침을 제출합니다.

---

## 기능

- SMS 본인인증 기반 설정 흐름
- 여러 가스 계정 동시 등록 지원 (계정별 별도 기기로 분리)
- 가스 계약 자동 조회 — 복수 계약이 있으면 직접 선택
- **당월 가스 청구금액** 센서 (요금 세부 내역 전체 속성)
- **전월 가스 청구금액** 센서 (이전 청구 이력 속성)
- **가스미터 검침 이력** 센서 (자가검침 등록 계정에만 생성)
- **자가검침 제출 결과** 바이너리 센서 (자가검침 등록 계정에만 생성)
- 자가검침 미등록 계정이 나중에 신청하면 관련 센서 자동 추가
- 매일 오전 8시 데이터 자동 갱신 + 자가검침 자동 제출
- 검침값 올림(ceil) / 내림(floor) 옵션
- 서비스 호출을 통한 수동 자가검침 제출

## 설치

### HACS (권장)

1. **HACS → Integrations** 로 이동합니다.
2. 우측 상단 메뉴 → **Custom repositories** 를 선택합니다.
3. 아래 주소를 입력하고 Category 를 **Integration** 으로 선택합니다.

   ```
   https://github.com/af950833/korea_gasapp
   ```

4. **Korea Gas App** 을 검색해 설치합니다.
5. Home Assistant를 재시작합니다.

### 수동 설치

`custom_components/korea_gasapp/` 폴더를 HA 설정 경로의 `custom_components/` 아래에 복사한 뒤 재시작합니다.

## 통합 추가

**설정 → 기기 및 서비스 → 통합 추가** 에서 **Korea Gas App** 을 선택합니다.

### 1단계 — 본인인증

| 항목 | 설명 |
|---|---|
| 이름 | 가스앱에서 사용하는 이름 |
| 휴대폰 번호 | 가스앱에 등록된 번호 |
| 주민등록번호 앞부분 | 앞 6자리 + 하이픈 + 성별 코드, 예: `950301-1` |
| 통신사 | SKT / KT / LG U+ (알뜰폰 포함) |

> 주민등록번호 앞부분은 설정 과정에서만 사용하며 저장하지 않습니다.

### 2단계 — SMS 인증번호 입력

문자로 받은 인증번호를 입력합니다.

### 3단계 — 자가검침 설정

| 항목 | 설명 |
|---|---|
| 가스미터 현재값 엔티티 | 자동 자가검침에 사용할 `input_number` / `number` / `sensor` 엔티티 |
| 검침값 반올림 방법 | **올림(ceil)** 또는 **내림(floor)** |

여러 계정을 추가하려면 통합 추가를 반복합니다. 각 계정은 HA에서 별도 기기로 표시됩니다.

## 엔티티

계약번호가 `1111111`인 경우:

| 엔티티 ID | 이름 | 아이콘 | 생성 조건 |
|---|---|---|---|
| `sensor.current_bill_charge_1111111` | 당월 가스 청구금액 | `mdi:gas-burner` | 항상 |
| `sensor.annual_bill_charge_1111111` | 전월 가스 청구금액 | `mdi:receipt-clock-outline` | 항상 |
| `sensor.indication_history_1111111` | 가스미터 검침 이력 | `mdi:meter-gas-outline` | 자가검침 등록 계정만 |
| `binary_sensor.gas_meter_submission_result_1111111` | 가스미터 자가검침 제출 결과 | 상태별 동적 | 자가검침 등록 계정만 |

상태값: 전월 청구금액 (원)


### 가스미터 검침 이력

상태값: 가장 최근 자가검침일 (날짜)

```yaml

202604 지침(m³): 447
202602 자가검침일: "2026년 2월 20일"
202602 지침(m³): 432
202512 자가검침일: "2025년 12월 22일"
202512 지침(m³): 417
```

### 자가검침 제출 결과

| 상태 | 의미 | 아이콘 |
|---|---|---|
| `on` | 마지막 제출 성공 | `mdi:check-circle-outline` |
| `off` | 마지막 제출 실패 또는 미제출 | `mdi:alert-circle-outline` |
| 알 수 없음 | 아직 시도 없음 | `mdi:help-circle-outline` |

속성:

| 속성 | 성공 | 실패 | 설명 |
|---|:---:|:---:|---|
| `last_attempt_at` | ✔ | ✔ | 마지막 시도 일시 |
| `reading` | ✔ | ✔ | 제출한 검침값 (m³) |
| `result_message` | ✔ | — | 성공 메시지 |
| `failure_reason` | — | ✔ | 실패 이유 |
| `source` | ✔ | ✔ | `auto` 또는 `manual` |

HA 재시작 후에도 마지막 결과가 유지됩니다.

## 업데이트 스케줄

매일 **오전 8시** 에 한 번 자동 실행됩니다.

1. 청구·검침 정보 전체 갱신
2. 자가검침 등록 계정이고 오늘이 `periodStart + 1일` 이면 자동 제출
3. 제출 결과를 바이너리 센서에 즉시 반영

`periodStart` 를 API에서 받지 못한 날은 자동 제출을 건너뜁니다.

---

## 자동 자가검침

자가검침이 등록된 계정은 가스앱 API의 `periodStart` 다음 날 오전 8시에 자동 제출합니다.

### 검침값 반올림

| 옵션 | 446.4 | 446.7 |
|---|---|---|
| 올림 (ceil) | 447 | 447 |
| 내림 (floor) | 446 | 446 |

## 옵션

**설정 → 기기 및 서비스 → Korea Gas App → 설정** 에서 변경 가능:

| 옵션 | 설명 |
|---|---|
| 가스미터 현재값 엔티티 | 자동 자가검침에 사용할 엔티티 |
| 검침값 반올림 방법 | 올림(ceil) 또는 내림(floor) |

---

## 수동 자가검침 서비스

```yaml
service: korea_gasapp.submit_meter_reading
data:
  reading: 447
```

여러 계정이 등록된 경우 `account` 를 지정합니다.

```yaml
service: korea_gasapp.submit_meter_reading
data:
  account: "우리집"   # 통합 제목 / 사용계약번호 / 고객번호 / 기기명 중 하나
  reading: 447
```

수동 제출 결과도 바이너리 센서에 반영됩니다.

---

## 자가검침 센서 자동 활성화

자가검침 미등록 계정은 관련 센서 없이 시작합니다. 가스앱에서 자가검침을 신청하면 다음 오전 8시 갱신 시점에 `indication_history` 센서와 `gas_meter_submission_result` 바이너리 센서가 자동 추가되며, 자동 제출도 활성화됩니다.

## 개인정보 및 보안

- 이름, 휴대폰 번호, 주민등록번호 앞부분, 인증번호는 설정 중에만 사용하며 저장하지 않습니다.
- 가스앱 세션 토큰과 계약 정보는 HA 설정 저장소에 저장됩니다.

---

## 참고 사항

- 공식 가스앱 API 문서 기반이 아닙니다. 내부 API 변경 시 동작이 실패할 수 있습니다.
- 자동 제출이 실패하면 `failure_reason` 속성에 이유가 기록됩니다. 수동 서비스로 재시도할 수 있습니다.
